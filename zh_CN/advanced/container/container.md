# 容器管理

容器是 Docker 的核心概念，是镜像的运行实例。 通过容器管理页面，你可以创建、启动、停止和管理容器。

## 容器列表

进入 **容器** 页面，默认显示容器列表。

![容器列表](/images/container/container-list.png)

列表显示以下信息：

- **容器名称**：容器的名称
- **状态**：运行状态开关
- **镜像**：容器使用的镜像
- **端口**：端口映射（主机端口 -> 容器端口）
- **运行状态**：详细的运行状态信息
- **操作**：终端、日志、重命名等

## 创建容器

点击 **创建容器** 按钮打开创建对话框。

![创建容器](/images/container/container-create.png)

### 基本设置

- **容器名称**：可选，留空则自动生成
- **镜像**：Docker 镜像名称，例如 `nginx`，`mysql:8.4`，`your_username/your_image:tag`
- **网络**：选择容器使用的网络
- **重启策略**：容器退出后的重启行为
  - 无：不自动重启
  - Always：总是重启
  - On failure：失败时重启（默认重试 5 次）
  - Unless stopped：除非手动停止，否则重启

本选项卡底部的 **容器选项** 区域提供以下开关：

- **TTY (-t)**：分配伪终端
- **STDIN (-i)**：保持标准输入打开
- **自动移除**：容器停止后自动删除
- **特权模式**：授予容器完整的系统权限（请谨慎使用）

### 端口被占用：修改映射端口

**端口模式** 提供两种选项：

- **映射端口**：手动将容器端口映射到主机端口
- **暴露全部**：自动将镜像中所有暴露的端口映射到随机主机端口

![端口映射](/images/container/container-create-port.png)

在 **映射端口** 模式下，每条规则包含一个可选的绑定 IP、一个主机端口范围（起始/结束）、一个容器端口范围（起始/结束）以及协议（TCP 或 UDP）。 若要映射单个端口，将起始值和结束值设为相同的数字即可。

例如，将主机端口 8080 映射到容器端口 80，即把主机起始/结束设为 `8080`，容器起始/结束设为 `80`。

### 卷挂载

将主机目录或数据卷挂载到容器中。 每个挂载由主机路径、容器路径和访问模式（**读写** 或 **只读**）组成。 主机目录请使用绝对路径。

例如，将主机的 `/opt/ace/data` 目录挂载到容器的 `/data` 目录。

### 资源限制

限制容器可以使用的资源。 将值设为 `0` 表示不限制：

- **内存 (MB)**：容器可使用的最大内存，单位为 MB
- **CPU 核心数**：容器可使用的 CPU 核心数（例如 `0.5` 表示半个核心）
- **CPU 权重**：相对 CPU 权重（默认 `1024`）；竞争时数值越高获得的 CPU 时间越多

### 环境

在 **环境** 选项卡中可以设置以下内容：

- **环境变量**：容器的环境变量，格式为 `KEY=VALUE`
- **启动命令**：覆盖镜像默认的 **命令** (CMD) 和 **入口点** (ENTRYPOINT)
- **容器标签**：附加到容器上的自定义标签，格式为 `KEY=VALUE`

## 容器操作

### 批量操作

勾选多个容器后，可以进行批量操作：

- **启动**：启动选中的容器
- **停止**：停止选中的容器
- **重启**：重启选中的容器
- **强制停止**：强制停止选中的容器
- **暂停**：暂停选中的容器
- **恢复**：恢复暂停的容器
- **删除**：删除选中的容器

### 单个容器操作

- **终端**：打开容器的终端，在容器内执行命令（仅在容器运行时可用）
- 查看容器日志
- **重命名**：修改容器名称
- **更多**：包含启动、停止、重启、强制停止、暂停、恢复和删除操作的下拉菜单

## 清理容器

点击 **清理容器** 可删除所有已停止的容器，释放系统资源。

:::warning 注意
清理操作不可恢复， 请确保已停止的容器不再需要。
:::
