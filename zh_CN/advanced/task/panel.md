# 面板任务

面板任务显示 AcePanel 执行的后台任务，如应用安装、环境安装等。

**任务** 页面有两个标签页：**计划任务**（默认显示）和 **面板任务**。 本页仅介绍 **面板任务**。 对于基于时间的任务（如定时备份或脚本），请参阅 [计划任务](./schedule.md)。

## 任务列表

进入 **任务** > **面板任务** 标签页查看面板任务列表。

![面板任务](/images/task/task-panel.png)

列表显示以下信息：

- **任务名称**：任务描述
- **状态**：等待中/运行中/已完成/失败
- **创建时间**：任务创建时间
- **完成时间**：任务结束时间
- **操作**：查看日志、删除

## 任务状态

| 状态  | 说明         |
| --- | ---------- |
| 等待中 | 任务已排队，尚未开始 |
| 运行中 | 任务正在执行     |
| 已完成 | 任务执行成功     |
| 失败  | 任务执行失败     |

## 常见任务类型

### 应用安装

安装应用时会创建安装任务：

- 安装应用 Nginx
- 安装应用 MySQL
- 安装应用 Docker

### 应用卸载

卸载应用时会创建卸载任务：

- 卸载应用 MariaDB
- 卸载应用 Redis

### 应用更新

更新已安装的应用时会创建更新任务：

- 更新应用 Gitea
- 更新应用 Grafana

### 环境安装

安装、卸载或更新运行环境时会创建任务：

- 安装环境 PHP 8.4
- 安装环境 Node.js 20
- 更新环境 Go 1.24

## 查看日志

点击 **日志** 按钮查看任务的详细执行日志，包括：

- 命令输出
- 错误信息

日志以实时流式传输，因此你可以跟踪正在运行任务的进度。 你也可以在日志窗口内清空当前日志。 当任务仍处于 **等待中** 状态时，**日志** 按钮不可用。

日志对于排查任务失败原因非常有用。

:::tip 提示
如果日志加载失败，请禁用浏览器中的所有广告拦截器。
:::

## 删除任务

只有已完成或失败的任务才能删除。 等待中和运行中的任务无法删除。

:::tip 提示
删除任务只是从列表中移除记录，不会影响已安装的应用或环境。
:::

## 任务失败处理

如果任务失败：

1. 点击 **日志** 查看错误信息
2. 根据错误信息排查问题
3. 解决问题后重新执行操作

常见失败原因：

- 网络问题导致下载失败
- 磁盘空间不足
- 依赖包缺失
- 端口被占用

### 重启后任务被标记为失败

当面板重启时（例如更新后或服务器重启后），任何仍处于 **运行中** 状态的任务都会被自动标记为 **失败**。 这可以防止任务在执行它们的进程被中断时永久卡在 **运行中** 状态。 如果你在重启后发现某个任务被标记为 **失败**，只需重新执行原来的操作即可。

### 清理卡住的任务

在极少数情况下，任务可能会卡在 **等待中** 或 **运行中** 状态，并阻塞同类型的新任务。 你可以从服务器命令行强制清空队列：

```bash
acepanel clear-task
```

这会将所有等待中和运行中的任务标记为 **失败**，从而解除队列阻塞。 详情请参阅 [CLI 工具](../../quickstart/cli.md)，请仅在指导下使用此操作。

## 注意事项

1. 安装任务可能需要较长时间，请耐心等待
2. 任务执行期间可以关闭浏览器
3. 如果任务长时间无响应，可以刷新页面查看状态
4. 建议定期清理已完成的任务记录
5. 在上一个任务仍处于等待中或运行中时提交相同的任务会被拒绝，请先等待上一个任务完成
