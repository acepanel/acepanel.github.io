# 資料庫管理

資料庫管理頁面用於建立、檢視與刪除資料庫。

## 資料庫清單

前往 **資料庫** 頁面。 資料庫會依類型組織成不同分頁：**MySQL**、**PostgreSQL**、**ClickHouse**、**MongoDB**、**SQLite**、**Elasticsearch**、**Redis**，以及 **使用者**（資料庫使用者）與 **伺服器**（資料庫伺服器）。 預設顯示 MySQL 分頁。

![資料庫清單](/images/database/database-list.png)

對於關聯式與文件型資料庫分頁（MySQL/PostgreSQL/ClickHouse/MongoDB/SQLite），清單會顯示以下資訊：

- **資料庫名稱**：資料庫名稱
- **伺服器**：所屬的資料庫伺服器
- **編碼**：字元編碼（僅 MySQL/PostgreSQL）
- **註解**：備註（僅 PostgreSQL）。 此欄位支援行內編輯——輸入值後點選其他位置（失去焦點）即可立即儲存
- **操作**：刪除

**建立資料庫** 按鈕僅在 MySQL、PostgreSQL、ClickHouse 與 MongoDB 分頁中可用。 SQLite 資料表在此處以唯讀方式列出，Elasticsearch/Redis 則透過各自的資料檢視進行管理。

**使用者** 與 **伺服器** 分頁透過各自的頁面進行管理；詳情請參閱 [資料庫使用者](./user.md) 與 [資料庫伺服器](./server.md)。

## 建立資料庫

1. 點選 **建立資料庫** 按鈕
2. 填寫設定：
   - **伺服器**：選擇資料庫伺服器
   - **資料庫名稱**：只能使用字母、數字、底線與連字號，且不能以數字開頭
   - **建立使用者**：開啟後會在建立資料庫的同時一併建立一個新使用者。 關閉時，您可以選擇性地輸入一個現有的 **授權使用者**，為其授予新資料庫的權限
   - **使用者名稱** / **密碼**：新使用者的憑證（僅當 **建立使用者** 開啟時；點選 **產生** 可取得隨機密碼）。 使用者名稱不能為 `root` 或 `admin`
   - **主機**：僅 MySQL 適用，選擇使用者允許的主機——**本機 (localhost)**、**全部 (%)** 或 **指定**（輸入自訂的主機位址）
3. 點選 **送出**

## 刪除資料庫

點選資料庫右側的 **刪除** 按鈕即可刪除該資料庫。 出現的確認對話方塊帶有 5 秒倒數計時；確認按鈕會在倒數計時結束後才能點選，讓您有機會中止意外的刪除操作。

:::danger 警告
刪除資料庫會永久刪除該資料庫中的所有資料。 此操作無法復原！ 請務必事先備份重要資料。
:::

## 資料庫命名規範

- 只能使用字母、數字、底線與連字號
- 不能以數字開頭
- 避免使用資料庫保留字
- 建議使用有意義的名稱，例如 `wordpress`、`myapp_production`

## 常見問題

### 建立資料庫失敗

- 檢查資料庫名稱是否符合命名規範
- 檢查是否已存在同名的資料庫
- 檢查資料庫服務是否正常執行

### 資料庫編碼問題

如果出現亂碼，請檢查：

1. 資料庫編碼是否正確
2. 連線時是否指定了正確的編碼
3. 應用程式的編碼設定
