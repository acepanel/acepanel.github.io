# 監控

監控模組用於記錄和查看伺服器的效能數據，包括 CPU、記憶體、磁碟 I/O 和網路流量。

## 監控頁面

![監控頁面](/images/monitor/monitor.png)

## 監控設定

### 啟用監控

開啟 **啟用監控** 開關後，系統會定期採集效能數據。

### 保存天數

設定監控數據的保留時間，預設 30 天。 超過保留時間的數據會自動清理。

### 清除監控記錄

點擊 **清除監控記錄** 按鈕可以手動清除所有歷史監控數據。

## 監控指標

### 負載（Load Average）

系統負載反映了 CPU 的繁忙程度：

- **1 分鐘負載**：最近 1 分鐘的平均負載
- **5 分鐘負載**：最近 5 分鐘的平均負載
- **15 分鐘負載**：最近 15 分鐘的平均負載

負載值的參考：

- 負載 < CPU 核心數：系統運行流暢
- 負載 = CPU 核心數：系統滿負荷運行
- 負載 > CPU 核心數：系統過載，可能出現卡頓

### CPU 使用率

顯示 CPU 的使用百分比，包括：

- 使用者態使用率
- 系統態使用率
- 空閒率

### 記憶體使用

顯示記憶體的使用情況：

- 已使用記憶體
- 可用記憶體
- 快取/緩衝區

### 磁碟 I/O

顯示磁碟的讀寫速度：

- 讀取速度（KB/s 或 MB/s）
- 寫入速度（KB/s 或 MB/s）

可以選擇要監控的磁碟裝置。

### 網路流量

顯示網路介面的流量：

- 傳送速度
- 接收速度

可以選擇要監控的網路介面。

## 時間範圍

每個監控圖表都支援選擇時間範圍：

- **昨天**：查看昨天的數據
- **今天**：查看今天的數據
- **近 7 天**：查看最近一週的數據
- **自訂**：選擇任意時間範圍

## 使用場景

### 效能分析

透過監控數據分析伺服器效能瓶頸：

- CPU 持續高負載：考慮最佳化程式或升級 CPU
- 記憶體不足：考慮增加記憶體或最佳化記憶體使用
- 磁碟 I/O 高：考慮使用 SSD 或最佳化資料庫查詢
- 網路頻寬不足：考慮升級頻寬

### 故障排查

出現問題時，查看歷史監控數據定位問題發生的時間和原因。

### 容量規劃

根據歷史數據趨勢，預測未來的資源需求，提前進行擴容。

## 注意事項

- 監控數據會佔用一定的磁碟空間
- 保存天數越長，佔用空間越大
- 建議根據實際需求設定合適的保存天數
