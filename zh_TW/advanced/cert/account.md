# 帳號管理

帳號管理頁面用於管理 ACME 帳號。 ACME 帳號用於向憑證授權機構申請憑證。

## 帳號清單

前往 **憑證** > **帳號** 標籤頁即可檢視帳號清單。

![帳號清單](/images/cert/cert-account.png)

清單會顯示下列資訊：

- **電子郵件**：帳號電子郵件
- **CA**：憑證授權機構
- **金鑰類型**：帳號金鑰類型
- **操作**：修改、刪除

## 建立帳號

1. 點選 **建立帳號** 按鈕
2. 填寫設定：
   - **CA**：選擇憑證授權機構
   - **金鑰類型**：選擇金鑰演算法
   - **電子郵件**：用於接收憑證相關通知
   - **KID**：僅當 CA 為 Google、LiteSSL 或 SSL.com 時才需要填寫
   - **HMAC**：僅當 CA 為 Google、LiteSSL 或 SSL.com 時才需要填寫
3. 點選 **提交**

:::tip 注意
LiteSSL、Google 與 SSL.com 需要先從其官方網站取得 EAB（KID 與 HMAC）。 僅當選擇上述其中一個 CA 時，才會顯示 KID 與 HMAC 欄位。 Google 在中國大陸無法存取，其他 CA 則視網路狀況而定，因此建議使用 Let's Encrypt 或 LiteSSL。
:::

### 憑證授權機構（CA）

| CA                      | 說明                            |
| ----------------------- | ----------------------------- |
| Let's Encrypt           | 最熱門的免費 CA，憑證有效期為 90 天         |
| GoogleCN                | 由 AcePanel 提供的 Google 憑證服務映像檔 |
| LiteSSL                 | 由 TrustAsia 提供的免費憑證服務         |
| ZeroSSL                 | 免費 CA，憑證有效期為 90 天             |
| SSL.com | 商業 CA，支援透過 EAB 進行 ACME 簽發     |
| Google                  | Google 的免費憑證服務                |

### 金鑰類型

| 類型      | 說明               |
| ------- | ---------------- |
| EC256   | 建議使用，金鑰較短，效能較佳   |
| EC384   | 安全性更高的 ECC 金鑰    |
| RSA2048 | 傳統的 RSA 金鑰，相容性最佳 |
| RSA4096 | 安全性更高的 RSA 金鑰    |

:::tip 建議
一般建議使用 EC256 金鑰，可兼顧安全性與效能。
:::

## 修改帳號

點選帳號右側的 **修改** 按鈕，即可修改 CA、金鑰類型、電子郵件、KID 與 HMAC。 儲存後會使用新設定向 CA 重新註冊帳號。

:::warning 注意
修改電子郵件後，憑證相關通知將會寄送至新的電子郵件。
:::

## 刪除帳號

點選帳號右側的 **刪除** 按鈕即可刪除該帳號。

:::warning 注意
刪除帳號後，使用該帳號申請的憑證將無法續約。 請先將憑證遷移至其他帳號，或先刪除相關憑證。
:::

## 帳號用途

ACME 帳號用於：

1. 向 CA 證明您的身分
2. 接收憑證到期提醒
3. 管理已申請的憑證
4. 撤銷憑證

## 多個帳號

您可以建立多個帳號：

- 不同的 CA 需要不同的帳號
- 可為不同的專案使用不同的帳號
- 方便管理與區分憑證
