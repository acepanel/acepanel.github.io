# 日誌清理

日誌清理功能用於掃描和清理系統及應用產生的日誌檔案，釋放磁碟空間。

![日誌清理](/images/toolbox/toolbox-log.png)

## 功能概覽

頁面頂部提供批次操作按鈕：

- **掃描所有**：掃描所有類型的日誌
- **清理所有**：清理所有已掃描的日誌

## 日誌類型

### 面板日誌

面板運行產生的日誌檔案。

- 點擊 **掃描** 檢查日誌大小
- 點擊 **清理** 刪除日誌檔案

### 網站日誌

網站的存取日誌和錯誤日誌，包括：

- Nginx 存取日誌（access.log）
- Nginx 錯誤日誌（error.log）
- 各網站的獨立日誌

:::tip 提示
網站日誌通常是佔用空間最大的日誌類型，建議定期清理。
:::

### MySQL 日誌

Percona/MySQL/MariaDB 資料庫的日誌檔案：

- 慢查詢日誌（slow query log）
- 二進位日誌（binlog）
- 錯誤日誌

:::warning 注意
二進位日誌用於資料恢復和主從複製，清理前請確認不再需要。
:::

### Docker

Docker 相關的日誌和未使用資源：

- 容器日誌
- 未使用的鏡像
- 建構快取

### 系統日誌

系統級別的日誌檔案：

- systemd journal 日誌
- `/var/log` 下的系統日誌
- 核心日誌

## 使用方法

1. 點擊各類型的 **掃描** 按鈕，查看日誌佔用空間
2. 確認要清理的日誌類型
3. 點擊 **清理** 按鈕刪除日誌

或者：

1. 點擊 **掃描所有** 一次性掃描所有日誌
2. 查看各類型的佔用情況
3. 點擊 **清理所有** 批次清理

## 注意事項

1. 清理操作不可恢復，請確認日誌不再需要
2. 重要日誌建議先備份
3. 生產環境建議保留最近的日誌用於問題排查
