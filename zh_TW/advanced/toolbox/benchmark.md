# 跑分測試

跑分功能用於測試伺服器的 CPU、記憶體和磁碟效能，協助你了解伺服器的實際效能表現。

![跑分測試](/images/toolbox/toolbox-benchmark.png)

## 測試項目

### CPU 測試

透過執行七項獨立的工作負載來評估處理器效能。 CPU 分數為所有子測試分數的總和：

- **影像處理**：產生一張影像並套用卷積（模糊）濾鏡
- **機器學習**：執行大型矩陣乘法
- **程式編譯**：計算大型費氏數列
- **AES 加密**：使用 AES-GCM 加密大型資料區塊
- **壓縮／解壓縮**：使用 gzip 壓縮與解壓縮資料
- **物理模擬**：執行 N 體重力模擬
- **JSON 解析**：序列化與反序列化大型 JSON 結構

### 記憶體測試

透過測量**記憶體頻寬**（單位 MB/s）與**記憶體存取延遲**（單位 ns）來評估記憶體子系統，並產生記憶體分數。

### 磁碟測試

透過直接 I/O 在三種區塊大小（**4KB**、**64KB** 與 **1MB**）下測量讀寫速度來評估儲存 I/O 效能，並產生磁碟分數。

## 使用方式

點選 **開始跑分** 按鈕，系統會依序執行 CPU 子測試、記憶體測試與磁碟測試，並顯示目前項目與整體進度。 測試完成後，會顯示 CPU、記憶體與磁碟的總分；將滑鼠移至分數上方即可檢視各項測試的詳細指標。

## 結果說明

跑分結果以數值形式呈現，數值越高表示效能越好。

:::warning 注意
跑分結果僅供參考。 由於系統資源排程、快取等因素的影響，測試結果可能與實際使用時的效能有所差異。
:::

## 測試建議

1. 測試前關閉不必要的服務與行程
2. 避免在業務尖峰時段進行測試
3. 多次測試取平均值會更準確
4. 不同時段的測試結果可能會有波動

## 效能最佳化參考

如果跑分結果不理想，可以考慮：

- **CPU 效能偏低**：檢查是否有行程佔用過高的 CPU
- **記憶體效能偏低**：檢查記憶體使用情況，是否存在記憶體洩漏
- **磁碟效能偏低**：檢查磁碟健康狀態，考慮升級為 SSD
