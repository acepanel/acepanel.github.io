# 系統

系統頁面提供伺服器基本設定的管理功能，包括 DNS、SWAP、主機名稱與時間設定。

## DNS

設定系統的 DNS 伺服器。

![DNS 設定](/images/toolbox/toolbox-system-dns.png)

頁面會顯示系統目前偵測到的 DNS 管理器。 如果管理器是 `resolv.conf`，面板會警告 DNS 的修改在系統重新啟動後將還原為預設值。

### 設定項目

- **DNS1**：主要 DNS 伺服器位址
- **DNS2**：次要 DNS 伺服器位址

儲存時 DNS1 與 DNS2 皆為必填項目。

## SWAP

管理系統的 SWAP 空間。

![SWAP 設定](/images/toolbox/toolbox-system-swap.png)

頁面會顯示目前的 SWAP 使用情況：

- 系統總計
- 已使用
- 閒置

### 設定面板 SWAP

設定由面板管理的 SWAP 檔案大小（單位：MB）。

- 設為 `0` 即停用面板 SWAP
- 建議依伺服器記憶體大小來設定，通常為記憶體的 1-2 倍

:::tip 提示
當實體記憶體不足時，SWAP 可提供額外的虛擬記憶體，但效能低於實體記憶體。 對於記憶體充足的伺服器，可能不需要啟用 SWAP。
:::

## 主機

設定系統主機名稱與 Hosts 檔案。

![主機設定](/images/toolbox/toolbox-system-host.png)

### 系統主機名稱

修改伺服器的主機名稱，例如 `myserver`、`web-01` 等。

### Hosts

編輯系統的 `/etc/hosts` 檔案，用於設定本機網域名稱解析。

常見用途：

- 封鎖特定網域
- 設定本機開發網域
- 加速特定網域的解析

## 時間

設定系統時區與時間同步。

![時間設定](/images/toolbox/toolbox-system-time.png)

### 選擇時區

從下拉清單中選擇伺服器的時區，例如 `Asia/Shanghai`。

### 修改時間

手動設定系統時間。

:::warning 注意
手動變更時間後，系統的自動時間同步仍可能覆寫您的設定。
:::

### NTP 伺服器

指定 **同步時間** 所使用的 NTP 伺服器。 這是供同步操作使用的一次性伺服器，不會被保留。 留空則會改用面板內建的伺服器（例如 `ntp.aliyun.com`、`ntp1.aliyun.com`、`ntp.tencent.com`、`time.windows.com`、`time.apple.com`），面板會自動偵測並挑選回應最快且可連線的一個。

點選 **設定預設伺服器** 開啟 **系統 NTP 伺服器設定** 對話框，您可以在其中管理系統長期保留的 NTP 伺服器清單。 該對話框會偵測系統使用的是 Chrony 還是 systemd-timesyncd，並將變更寫入其設定檔。 您可以新增或移除多個伺服器，點選 **重設為預設值** 還原內建清單，並點選 **儲存** 套用。 至少需要一個伺服器。

如果面板無法辨識時間服務，對話框會顯示警告，提示無法偵測到 NTP 服務，並要求您確認已安裝 Chrony 或 systemd-timesyncd。 在這種情況下，請先安裝其中一項服務，再保留 NTP 伺服器清單。

### 操作按鈕

- **儲存**：儲存時區與手動設定的時間
- **同步時間**：立即使用 NTP 伺服器（若留空則使用內建伺服器）同步系統時間
